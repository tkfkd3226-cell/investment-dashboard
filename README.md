# 투자 성과 대시보드

개인 투자 현황을 날짜별로 조회하는 정적 HTML 대시보드입니다.  
삼성증권 계좌1, 삼성증권 계좌2 실현분, 토스증권 실현분, 퇴직연금 데이터를 선택일 기준으로 조합해 표시합니다.

## 화면 구조

현재 주요 화면 순서는 다음과 같습니다.

1. 상단 투자 성과 대시보드
2. 연금+계좌 성과
3. 퇴직연금 현황
4. 증권계좌 현황
5. 투자 기간 분석
6. 계좌별 성과 요약
7. 장부상 결과물 VS 실제 보유액
8. 증권계좌 보유분
9. 원천별 투입/회수 요약

## 섹션별 기준

### 연금+계좌 성과

퇴직연금과 증권계좌를 합산한 최상위 요약 표입니다.

| 행 | 의미 |
|---|---|
| 퇴직연금 | 퇴직연금 평가금액, 납입원금, 운용수익 |
| 증권계좌 | 계좌1 + 계좌2 실현분 + 토스증권 실현분 |
| 합산 | 퇴직연금 + 증권계좌 |

### 퇴직연금 현황

퇴직연금 단독 현황입니다.

표시 항목:

- 퇴직연금 평가금액
- 퇴직연금 납입원금
- 퇴직연금 운용수익
- 퇴직연금 누적수익률
- 상품별 현황
- 전일 대비 변동

현금성자산 기준:

- 2026-06-30 기준 현금성자산 평균단가, 매수원금, 평가금액은 39,717원으로 정리
- 기업적립금 추가분은 현금성자산의 평균단가, 매수원금, 평가금액에 함께 반영
- 따라서 기업적립금 추가 자체로는 퇴직연금 운용수익을 증가시키지 않음

### 증권계좌 현황

증권계좌 단독 장부 기준 현황입니다.

포함 범위:

- 계좌1
- 2026-05-22 이후 계좌2 실현분
- 토스증권 실현분

퇴직연금은 이 섹션에 포함하지 않습니다.

### 증권계좌 보유분

선택일 현재 삼성증권 계좌1 보유종목과 증권계좌 현금을 표시합니다.

계좌1 투자원금과 보유종목 합계 원금은 기준이 다를 수 있습니다.

- 보유종목 합계 원금: 선택일 현재 보유종목 원가 기준
- 계좌1 투자원금: 실현손익, 원금회수, 재투입, 장부 조정 반영 기준

### 장부상 결과물 VS 실제 보유액

2026-06-18 이후 표시합니다.

```text
현재 증권계좌 및 현금 보유액(B)
= 현재 증권계좌 평가총액 + 계좌 밖 현금 2,035,097원

차액(A-B)
= 장부상 증권계좌 투자 결과물(A) - 현재 증권계좌 및 현금 보유액(B)
```

차액은 수익실현분 카드대금 사용액으로 정리합니다.

## 퇴직연금 기업적립금 추가

### 파일 구조

기업적립금은 `data/pension_contributions.json`에 저장합니다.

```json
{
  "meta": {
    "description": "퇴직연금 기업적립금 추가 기록. 2026-06-30까지의 기 반영분은 portfolio.json의 pensionContributionPrincipal에 포함된 것으로 보고, 2026-07 이후 신규 적립분만 기록.",
    "basePrincipalIncludesThrough": "2026-06-30",
    "monthlyDefaultAmount": 618060,
    "annualAdjustmentMonth": 2,
    "note": "매월 기본 618,060원. 매년 2월 금액 변경 가능."
  },
  "contributions": []
}
```

추가 항목 예시:

```json
{
  "date": "2026-07-01",
  "source": "company",
  "amount": 618060,
  "memo": "2026년 7월 기업적립금"
}
```

### 화면 사용법

페이지 상단 우측의 톱니바퀴 버튼을 클릭하면 `퇴직연금 기업적립금 추가` 모달이 열립니다.

입력 항목:

- 일자
- 금액
- 메모
- 저장 PIN

버튼:

| 버튼 | 기능 |
|---|---|
| 저장 | Netlify Function을 통해 GitHub의 `pension_contributions.json` 자동 수정 |
| 추가 JSON 만들기 | 수동 백업용 JSON 조각 생성 |
| 복사 | 생성된 JSON을 클립보드에 복사 |
| 선택 항목 삭제 | 등록된 기업적립금 내역에서 선택한 항목 삭제 |

같은 날짜와 같은 source 값의 항목이 이미 있으면 새로 추가하지 않고 기존 항목을 수정합니다.  
테스트로 넣은 항목을 되돌릴 때는 금액을 0원으로 저장하지 않고, 모달의 `등록된 기업적립금 내역`에서 해당 항목을 선택한 뒤 `선택 항목 삭제`를 사용합니다.

### 자동 저장 흐름

```text
대시보드 저장 버튼
→ /.netlify/functions/save-pension-contribution
→ GitHub API로 data/pension_contributions.json 읽기
→ contributions 배열에 신규 추가 또는 기존 수정
→ GitHub 커밋 생성
→ Netlify 자동 재배포
```

## Netlify 환경변수

자동 저장 기능을 사용하려면 Netlify에 다음 환경변수를 설정합니다.

| 환경변수 | 필수 | 설명 |
|---|---|---|
| `GITHUB_TOKEN` | 필수 | GitHub Personal Access Token |
| `GITHUB_OWNER` | 필수 | GitHub 사용자명 또는 조직명 |
| `GITHUB_REPO` | 필수 | 저장소 이름 |
| `GITHUB_BRANCH` | 권장 | 기본값 `main` |
| `ADMIN_PIN` | 권장 | 저장 버튼 사용 시 입력할 PIN |

`ADMIN_PIN`을 설정하지 않으면 PIN 없이 저장이 가능하므로 운영 중에는 설정하는 것을 권장합니다.

## Netlify Function

파일 위치:

```text
netlify/functions/save-pension-contribution.js
```

역할:

- POST 요청만 허용
- `ADMIN_PIN` 검증
- 입력값 검증
- GitHub에서 `data/pension_contributions.json` 조회
- 같은 날짜/source가 있으면 수정
- 없으면 추가
- 날짜순 정렬 후 GitHub에 커밋

## GitHub Token 만료 시 처리

GitHub Personal Access Token에 만료일을 설정한 경우, 만료 후에는 자동 저장이 실패합니다.

현재 GitHub 토큰은 2026-07-01에 90일 만료로 생성했으므로, 2026-09-28까지 새 토큰으로 교체하는 것을 권장합니다.  
미교체 시 2026-09-29 전후부터 `퇴직연금 기업적립금 추가` 저장 버튼에서 GitHub 인증 실패/저장 실패가 발생할 수 있습니다.

처리 순서:

1. GitHub에서 새 Fine-grained token 생성
2. 기존과 동일하게 해당 repo의 `Contents: Read and write` 권한 부여
3. Netlify 환경변수 `GITHUB_TOKEN` 값을 새 토큰으로 교체
4. Netlify 재배포
5. 대시보드 저장 테스트

이 내용은 화면 상단 우측 톱니바퀴 → `퇴직연금 기업적립금 추가` 모달 하단에서도 확인할 수 있습니다.

## 운영 메모

- 2026-06-30까지의 퇴직연금 납입원금은 `portfolio.json`의 `pensionContributionPrincipal`에 이미 반영된 것으로 봅니다.
- `pension_contributions.json`에는 2026-07 이후 신규 기업적립금만 기록합니다.
- 기업적립금은 퇴직연금 납입원금과 현금성자산 평가금액에 동시에 반영합니다.
- 매월 기본 기업적립금은 618,060원입니다.
- 매년 2월 금액이 변경될 수 있습니다.
- 자동 저장 실패 시 모달에서 JSON을 생성해 수동으로 `pension_contributions.json`에 추가할 수 있습니다.
