# 투자 대시보드 최종본

최종 기준 파일명: `investment-dashboard-main_final_menu_fix.zip`

삼성증권 일반 증권계좌, 퇴직연금 계좌, 토스 현금성 자산을 함께 보는 정적 투자 대시보드입니다.  
기존 단일 HTML 구조에서 `index.html`, `style.css`, `script.js`로 분리되어 있으며, Netlify 정적 배포와 GitHub Actions 가격 업데이트 구조를 유지합니다.

---

## 1. 최종 반영 요약

### 1-1. 파일 구조 정리

- `index.html`에서 CSS/JS를 인라인으로 두지 않고 외부 파일로 분리했습니다.
- `style.css`는 화면 스타일과 모바일 QA 보정 CSS를 포함합니다.
- `script.js`는 데이터 로딩, 계산, 렌더링, 차트, 모달 동작을 담당합니다.
- `data/portfolio.json`, `data/prices.json`, `data/pension_contributions.json` 구조를 유지합니다.
- `.github/workflows/update-prices.yml`, `scripts/update_prices.py`, `netlify/functions/save-pension-contribution.js` 구조를 유지합니다.

### 1-2. 퇴직연금 현황 추가/보정

- 퇴직연금 현황 아래에 투자 기간 분석 차트 3개를 추가했습니다.
  - 운용수익 및 누적수익률
  - 연금상품별 운용수익
  - 평가액 비중
- 퇴직연금 차트는 `2026-06-04`부터 시작합니다.
- `2026-06-04`처럼 전일 퇴직연금 데이터가 없는 날짜는 `전일 데이터가 없습니다.`만 표시합니다.
- `2026-06-17`, `2026-06-25`처럼 `display:false`인 날짜도 퇴직연금 데이터가 있으면 전일 대비 변동을 표시합니다.
- 퇴직연금 차트 Y축은 실제 최소/최대값을 그대로 쓰지 않고 보기 좋은 단위의 눈금으로 보정했습니다.
- 퇴직연금 평가액 비중 차트와 하단 카드에서 현금성자산 카드는 제외했습니다.
- 현재 평가총액 카드에는 현금 포함 문구를 작은 글씨로 표시합니다.

### 1-3. 증권계좌 현황 정리

- 증권계좌 현황 내부의 `증권계좌 요약 / YYYY-MM-DD 기준` 제목줄을 제거했습니다.
- KPI 카드 4개는 유지합니다.
  - 증권계좌 투자 결과물
  - 기준 투입원금
  - 총 합산 누적손익
  - 투자대비 이익률
- `종목별 누적 손익` 문구는 `종목별 누적손익`으로 통일했습니다.

### 1-4. 모바일 메뉴 정리

모바일 우측 상단 `☰` 빠른 이동 메뉴를 최종 화면 제목 기준으로 정리했습니다.

- 퇴직연금 현황
  - 상품별 현황
  - 전일 대비 변동
  - 운용수익 및 누적수익률
  - 연금상품별 운용수익
  - 평가액 비중
- 증권계좌 현황
  - 계좌별 성과 요약
  - 누적손익 및 누적수익률
  - 종목별 누적손익
  - 평가액 비중
  - 증권계좌 보유분
  - 장부상 결과물 VS 실제 보유액

### 1-5. 표기 통일

- `KODEX삼전채권혼합` → `KODEX 삼전채권혼합`
- `KODEX200` → `KODEX 200`

표, 카드, 차트 범례, tooltip, 모바일 메뉴 등 화면 표시명을 기준으로 통일했습니다.

## 3. 데이터 파일

### 3-1. `data/portfolio.json`

투자 포트폴리오 기본 정보를 담습니다.

- 증권계좌 보유 종목
- 퇴직연금 보유 상품
- 보유 수량
- 원가
- 표시용 클래스

퇴직연금 상품명 최종 표기:

```text
KODEX 200TR
KODEX AI반도체
KODEX 삼전채권혼합
```

증권계좌 ETF 표기:

```text
KODEX 200
```

### 3-2. `data/prices.json`

날짜별 가격과 현금성 데이터를 담습니다.

퇴직연금 가격 데이터는 `2026-06-04`부터 `2026-07-01`까지 보강되어 있습니다.  
`display:false`는 화면 날짜 선택 노출 여부에 영향을 줄 수 있으나, 퇴직연금 전일 대비 변동 표시 여부는 퇴직연금 데이터 존재 여부를 기준으로 판단합니다.

### 3-3. `data/pension_contributions.json`

퇴직연금 기업적립금 추가 입력 내역을 담습니다.  
Netlify Function을 통해 저장하는 구조를 유지합니다.

## 4. 화면 구성

### 4-1. 투자 요약

전체 평가액, 전체 투자원금, 전체 누적손익, 수익률 등 통합 KPI를 표시합니다.

### 4-2. 퇴직연금 현황

퇴직연금 영역은 다음 순서로 표시됩니다.

```text
퇴직연금 KPI
상품별 현황
전일 대비 변동
투자 기간 분석
  - 운용수익 및 누적수익률
  - 연금상품별 운용수익
  - 평가액 비중
```

#### 전일 대비 변동

- 전일 데이터가 없으면 `전일 데이터가 없습니다.`만 표시합니다.
- 전일 데이터가 있으면 KPI, 표, 모바일 카드형 목록을 표시합니다.

#### 운용수익 및 누적수익률

- 막대: 운용수익, 전일대비손익
- 선: 누적수익률
- 하단 카드:
  - 최종 운용수익
  - 최종 누적수익률
  - 최고 운용수익
  - 최대 낙폭
  - 최고의 하루
  - 최악의 하루

#### 연금상품별 운용수익

- 3개 퇴직연금 상품의 운용수익 변화를 표시합니다.
- 웹에서는 상품별 카드 3개가 한 줄 3열로 표시됩니다.
- 변동률은 전일 운용수익 절댓값 대비로 계산합니다.

#### 평가액 비중

- 차트와 카드에서 현금성자산은 제외합니다.
- 현재 평가총액 카드에 현금 포함 금액을 작은 글씨로 표시합니다.

예시:

```text
56,760,222원
(현금 39,717원 포함)
```

### 4-3. 증권계좌 현황

증권계좌 영역은 다음 순서로 표시됩니다.

```text
증권계좌 KPI
계좌별 성과 요약
투자 기간 분석
  - 누적손익 및 누적수익률
  - 종목별 누적손익
  - 평가액 비중
증권계좌 보유분
장부상 결과물 VS 실제 보유액
```

증권계좌 내부 제목줄 `증권계좌 요약 / YYYY-MM-DD 기준`은 제거되어 있습니다.

## 5. 차트 보정

퇴직연금 차트의 Y축 라벨은 실제 데이터 최소/최대값을 그대로 출력하지 않습니다.

예를 들어 아래처럼 애매한 값이 그대로 찍히지 않도록 했습니다.

```text
18,433,283
8,533,928
-1,365,428
```

대신 보기 좋은 단위의 tick을 생성합니다.

```text
20,000,000
15,000,000
10,000,000
5,000,000
0
-5,000,000
```

관련 함수:

```text
niceStep()
niceTickInfo()
```

## 7. 기업적립금 추가 모달

`퇴직연금 기업적립금 추가` 버튼으로 모달을 열 수 있습니다.

모달 닫기 시 다음 경고가 발생하지 않도록 포커스 처리를 추가했습니다.

```text
Blocked aria-hidden on an element because its descendant retained focus
```

닫을 때 모달 내부 포커스를 blur 처리하고, 상단 버튼으로 포커스를 복귀합니다.
