# 투자 성과 대시보드

개인 투자 현황을 날짜별로 조회하는 정적 HTML 대시보드입니다.  
삼성증권 계좌1 일별 복원 데이터, 계좌2 실현분, 토스증권 실현분, 퇴직연금 데이터를 선택일 기준으로 조합해 표시합니다.

## 현재 구조 요약

| 구분 | 기준 |
|---|---|
| 2026-03-23 ~ 2026-06-25 | `data/account1_daily_snapshots.json`의 삼성증권 계좌1 원시 복원 데이터 |
| 2026-06-26 이후 | `data/prices.json`의 자동 업데이트 가격 데이터 |
| 계좌2 | 2026-05-21까지 제외, 2026-05-22 전량 매도 이후 실현분 포함 |
| 토스증권 | 2026-03-23 이전 실현 완료 거래로 2026-03-23 이후 전 구간 포함 |
| 퇴직연금 | 데이터가 있는 날짜에만 퇴직연금 현황 및 계좌+연금 성과 표시 |
| 장부상 결과물 검산 | 2026-06-18 이후 표시 |
| 계좌 밖 현금 | 2026-06-18 확인값 2,035,097원 유지 |

## 주요 기능

- 월/일 선택 UI
- 선택일 기준 화면 전체 재계산
- 삼성증권 계좌1 보유종목 및 현금 표시
- 증권계좌 평가액 비중 표시
- 투자 기간 분석 차트
- 핵심종목별 기여도 차트
- 계좌별 성과 요약
- 퇴직연금 현황
- 계좌+연금 성과
- 장부상 결과물 VS 실제 보유액 검산
- 모바일 차트 툴팁 및 표 가로 스크롤 지원

## 날짜별 데이터 기준

### 2026-03-23 ~ 2026-06-25

`data/account1_daily_snapshots.json` 기준입니다.

이 파일은 과거 원시데이터를 복원한 삼성증권 계좌1 일별 스냅샷입니다.

포함 항목:

- 기준일
- 보유종목
- 수량
- 평단
- 현재가
- 투자원금
- 평가금액
- 평가손익
- 손익률
- 증권계좌 현금
- 계좌1 합계

### 2026-06-26 이후

`data/prices.json` 기준입니다.

현재 `data/portfolio.json`의 보유수량과 자동 업데이트된 가격을 조합해 평가금액과 손익을 계산합니다.

### `data/performance_snapshots.json`

기존 자동 업데이트 누적 기록 파일입니다.  
불필요 파일이 아니므로 삭제하지 않습니다.

## 계좌 포함 기준

| 항목 | 포함 기준 |
|---|---|
| 삼성증권 계좌1 | 전 구간 포함 |
| 삼성증권 계좌2 | 2026-05-21까지 제외, 2026-05-22 전량 매도 이후 실현분 포함 |
| 토스증권 | 2026-03-23 이전 실현 완료 거래이므로 2026-03-23 이후 전 구간 포함 |
| 퇴직연금 | 가격 데이터가 있는 날짜에 표시 |

계좌2가 제외되는 날짜에는 계좌별 성과 요약 표에서 계좌2 행을 숨기고, 표 하단에 제외 사유를 표시합니다.

## 퇴직연금 표시 기준

퇴직연금은 중간 매도, 재매수, 리밸런싱, 기업적립금 추가 이력이 있어 데이터가 있는 날짜만 표시합니다.

현재 주요 표시 기준일:

- 2026-06-18
- 2026-06-26
- 2026-06-29
- 2026-06-30
- 이후 자동 업데이트로 생성되는 날짜

`prices.json`의 `display:false` 데이터는 전일 대비 계산용 보조 데이터일 수 있으므로 유지합니다.  
단, 해당 날짜를 직접 선택했을 때는 퇴직연금 현황 안의 `전일 대비 변동` 섹션을 숨길 수 있습니다.

## 화면별 계산 기준

### 상단 투자 성과 대시보드

선택일의 포함 범위에 따라 계산합니다.

| 구간 | 계산 기준 |
|---|---|
| 2026-03-23 ~ 2026-05-21 | 계좌1 + 토스증권 실현분 |
| 2026-05-22 이후 | 계좌1 + 토스증권 실현분 + 계좌2 실현분 |
| 퇴직연금 데이터 있는 날 | 위 기준 + 퇴직연금 성과 |

### 증권계좌 보유분

선택일 현재 삼성증권 계좌1 기준입니다.

현금 처리:

- 증권계좌 현금은 투자원금에 포함
- 증권계좌 현금은 평가금액에 포함
- 증권계좌 현금의 평가손익은 0원
- 총계 손익률 = 보유종목 평가손익 ÷ 보유분+현금 투자원금

예: 2026-06-30 기준

| 항목 | 금액 |
|---|---:|
| 보유종목 합계 원금 | 16,282,745원 |
| 증권계좌 현금 | 17,255원 |
| 총계 원금 | 16,300,000원 |

계좌1 투자원금과 보유종목 합계 원금은 기준이 다를 수 있습니다.

- 보유종목 합계 원금: 현재 보유종목 원가 기준
- 계좌1 투자원금: 실현손익, 원금회수, 재투입, 장부 조정 반영 기준

### 평가액 비중

현재 증권계좌 평가총액 기준입니다.

```text
현재 증권계좌 평가총액 = ETF 평가액 + 개별주식 평가액 + 증권계좌 현금
```

계좌 밖 현금, 장부상 전체 투자 결과물, 수익실현분 카드대금 사용액과 섞지 않습니다.

### 투자 기간 분석 차트

계좌1 보유분 중심의 누적손익과 일별손익을 표시합니다.  
상단 총 합산 누적손익은 토스증권 실현분, 2026-05-22 이후 계좌2 실현분, 데이터가 있는 날의 퇴직연금이 포함될 수 있어 차트 값과 다를 수 있습니다.

### 핵심종목별 기여도

아래 핵심종목 기준으로 표시합니다.

- 삼성전자
- SK하이닉스
- 현대차
- KODEX 200

후성, KoAct, AI반도체, 로봇액티브 등은 현재 핵심종목 기여도 카드에서 제외합니다.

### 장부상 결과물 VS 실제 보유액

2026-06-18 이후 표시합니다.

계산 기준:

```text
현재 증권계좌 및 현금 보유액(B) = 현재 증권계좌 평가총액 + 계좌 밖 현금 2,035,097원
차액(A-B) = 장부상 전체 투자 결과물(A) - 현재 증권계좌 및 현금 보유액(B)
```

계좌 밖 현금은 2026-06-18 확인값 2,035,097원을 유지합니다.  
차액은 수익실현분 카드대금 사용액으로 정리합니다.


## 퇴직연금 기업적립금 데이터

`data/pension_contributions.json` 파일로 2026-07 이후 신규 기업적립금을 관리합니다.

현재 전제:

```text
2026-06-30까지의 기 반영분
= portfolio.json의 pensionContributionPrincipal에 이미 포함
```

따라서 `pension_contributions.json`은 2026-07 이후 기업적립금부터 기록합니다.

기본 설정:

```text
월 기본 기업적립금: 618,060원
금액 변경 가능 시점: 매년 2월
```

초기 파일 구조:

```json
{
  "meta": {
    "basePrincipalIncludesThrough": "2026-06-30",
    "monthlyDefaultAmount": 618060,
    "annualAdjustmentMonth": 2
  },
  "contributions": []
}
```

기업적립금 추가 예시:

```json
{
  "date": "2026-07-01",
  "source": "company",
  "amount": 618060,
  "memo": "2026년 7월 기업적립금"
}
```

계산 기준:

```text
선택일 기준 퇴직연금 납입원금
= portfolio.json의 pensionContributionPrincipal
+ pension_contributions.json에서 선택일 이하 기업적립금 누적액
```

반영 위치:

- 퇴직연금 납입원금
- 퇴직연금 운용수익
- 퇴직연금 누적수익률
- 계좌+연금 성과

## 자동 업데이트

GitHub Actions가 평일 장 마감 이후 가격 데이터를 업데이트합니다.

| 파일 | 역할 |
|---|---|
| `.github/workflows/update-prices.yml` | 자동 업데이트 워크플로우 |
| `scripts/update_prices.py` | KRX 가격 조회 및 데이터 갱신 스크립트 |
| `data/prices.json` | 날짜별 가격 데이터 |
| `data/performance_snapshots.json` | 자동 업데이트 누적 성과 기록 |

현재 자동 업데이트는 기존 구조를 유지합니다.

```text
업데이트 대상:
- data/prices.json
- data/performance_snapshots.json
```

`data/account1_daily_snapshots.json`은 과거 복원 데이터이므로 자동 업데이트 대상이 아닙니다.

## 주의사항

- 날짜별 과거 복원은 삼성증권 계좌1 원시데이터 중심입니다.
- 퇴직연금은 별도 매매·리밸런싱·기업적립금 데이터가 필요해 전체 과거 복원 대상이 아닙니다.
- 계좌2 매도 전 상세 원시데이터는 해지한 펀드라 복원 대상이 아닙니다.
- README는 계산 기준 설명용 문서이며, 실제 계산은 `index.html`과 `data/*.json` 기준입니다.
